## AI 开发代理指令

### 1. 项目概述

* **目标**：将代码逻辑转化为小白可理解的“生活化比喻”与“拟人化群聊”。
* **核心功能**：代码解析、架构可视化、业务流程剧本化。

### 2. 开发规范

* **架构**：React (前端) + FastAPI (后端) + LLM/RAG (AI 层)。
* **数据流**：代码分片 -> 向量检索 -> 业务摘要 -> JSON 剧本。
* **AI 要求**：强制 JSON 格式输出，长代码必须先摘要再处理。

### 3. 代码风格

* **通用**：变量名自解释，逻辑清晰。
* **前端**：React Hooks + TypeScript，组件与业务逻辑分离。
* **后端**：Python PEP 8 规范，必须包含类型注解 (Type Hints)。

### 4. 测试要求

* **单元测试**：覆盖代码解析核心逻辑。
* **健壮性**：必须校验 AI 生成的 JSON 合法性，处理空目录及超大文件。
* **性能**：AI 解析需支持异步处理与进度回传。

### 5. 注意事项

* **安全**：严禁硬编码 API Key，严禁持久化存储用户源码。
* **文案**：禁止使用技术黑话，必须转化为生活化类比。
* **性能**：前端图表渲染不得阻塞主线程，控制大文件处理时延。
