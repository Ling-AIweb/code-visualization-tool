# 产品需求文档 (PRD)：代码逻辑可视化工具

## 1. 产品概述与目标用户

* **产品定位**：一款将复杂代码库转化为“大白话”和“拟人化剧本”的 AI 可视化解析工具。
* **核心价值**：打破技术壁垒，让产品经理、运营、小白创业者无需理解语法也能看懂业务逻辑。
* **目标用户**：
* **小白 PM/运营**：需要理解后端逻辑以制定业务规则。
* **零基础初学者**：通过生活化类比快速跨越编程“入坟”期。
* **非技术合伙人**：需要审计代码进度或评估技术架构的合理性。

## 3. 功能优先级 (Roadmap)

### 第一阶段：MVP (最小可行性产品)

* **重点**：跑通“代码 -> 白话解释 -> 静态群聊剧本”的流程。
* **功能**：单文件/小文件夹上传、AI 生成文本解释、静态 Mermaid 结构图、预设模板的群聊剧本。

### 第二阶段：V1.5 增强版

* **重点**：交互化与深度解析。
* **功能**：GitHub 仓库直接导入、动态群聊演示（逐条弹出）、交互式拓扑图（点击可下钻）。

### 第三阶段：V2.0 智能化与协作

* **重点**：多端适配与实时答疑。
* **功能**：AI 实时对话（用户追问“这段代码报错了怎么解释？”）、团队协作分享链接。

## 5. 技术栈建议

* **前端**：**React.js** 或 **Vue3** + **Tailwind CSS**（实现响应式与现代化 UI）。
* **图表库**：**Mermaid.js** 或 **React Flow**（用于生成可交互的逻辑架构图）。
* **后端**：**Python (FastAPI)** 或 **Node.js**。Python 在处理 AI 模型调用上更具优势。
* **大模型 (LLM)**：**Gemini 1.5 Pro** 或 **GPT-4o** 或其他模型。利用其超长上下文窗口来解析整个文件夹。
* **向量数据库**：**ChromaDB** 或 **Pinecone**。通过 RAG（检索增强生成）技术，在处理超大型项目时只提取核心逻辑，降低 Token 消耗。

## 6. 非功能性需求

* **性能**：10MB 以内的代码包，解析并生成剧本的时间应控制在 15 秒内。
* **安全性**：
* **源码保护**：明确告知用户代码仅用于分析，不进行存储，或提供解析后立即销毁的功能。
* **隐私限制**：自动脱敏代码中的 API Key、密码等敏感信息。


* **易用性**：提供“新手引导动画”，模拟一个 30 秒的“代码变群聊”过程。
